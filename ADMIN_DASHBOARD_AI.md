# 📊 AI-Driven Admin Dashboard — Острог Разом

## Огляд

Нова функціональність **AI-Driven Admin Dashboard** надає адміністраторам платформи «Острог Разом» потужний аналітичний інструмент з інтегрованим AI-помічником для аналізу даних про міські проблеми.

---

## 🆕 Що нового

### 1. Інтерактивна панель статистики
- **Карточки KPI**: Загальна кількість проблем, нові, виконані, критичні — з анімованими лічильниками
- **Тренди**: Графік створених vs. вирішених проблем за останні 6 місяців (AreaChart)
- **Статуси**: Інтерактивна кільцева діаграма розподілу за статусами
- **Категорії**: Горизонтальна стовпчикова діаграма проблем за категоріями
- **Пріоритети**: Секторна діаграма розподілу за пріоритетами
- **Останні проблеми**: Список 5 останніх створених проблем з посиланнями
- **Топ за рейтингом**: Список 5 проблем з найвищим рейтингом

### 2. AI-Аналітик (Вбудований чат)
- **Спеціалізований AI**: Chatbot, оптимізований для аналітики та статистики
- **Повний доступ до даних**: AI має доступ до всієї статистики системи в реальному часі
- **Швидкі запити**: Набір попередньо визначених кнопок для типових аналітичних запитів
- **Фільтрація через AI**: Можливість змінювати фільтри дашборду через запити до AI
- **Markdown-відповіді**: AI форматує відповіді з заголовками, списками та емодзі
- **Рекомендації**: AI може давати конкретні поради щодо покращення роботи

### 3. Додаткові можливості
- **Автоматичне оновлення**: Статистика оновлюється кожні 60 секунд
- **Responsive дизайн**: Адаптивний макет для різних розмірів екранів
- **Анімації**: Плавні переходи, hover-ефекти, анімовані лічильники
- **Фільтрація AI**: Коли адмін просить "покажи критичні проблеми", діаграми фільтруються відповідно

## 🏗️ Архітектура

### Backend (C# / .NET 8)

#### Нові методи в `IGeminiService`:
```csharp
Task<DashboardStatistics> GetDashboardStatisticsAsync(CancellationToken cancellationToken);
Task<AdminChatResponse> ProcessAdminChatMessageAsync(string message, CancellationToken cancellationToken);
```

#### Нові DTO:
- `DashboardStatistics` — повна статистика системи (20+ полів)
- `AdminChatResponse` — відповідь AI з опціональним фільтром
- `DashboardFilter` — фільтр, який AI може запропонувати
- `CategoryStat`, `StatusStat`, `PriorityStat`, `MonthlyTrend` — допоміжні типи

#### Нові API ендпоінти:
| Метод | URL | Опис | Авторизація |
|-------|-----|------|-------------|
| GET | `/chat/admin/statistics` | Отримати статистику дашборду | Administrator |
| POST | `/chat/admin/message` | Відправити повідомлення AI-аналітику | Administrator |

#### Файли змінені:
1. `api/src/Application/Services/GeminiService/IGeminiService.cs` — додані нові методи та типи
2. `api/src/Infrastructure/Services/GeminiService.cs` — реалізація `GetDashboardStatisticsAsync` та `ProcessAdminChatMessageAsync`
3. `api/src/API/Controllers/ChatController.cs` — нові ендпоінти для адмін-дашборду
4. `api/src/API/DTOs/Chat/ChatDtos.cs` — нові DTO для адмін-функціональності

### Frontend (React + TypeScript)

#### Нові компоненти:
- `AdminDashboardPage` — головна сторінка дашборду з графіками та чатом
- `StatCard` — картка статистики з анімацією
- `MiniStatCard` — компактна картка статистики
- `AnimatedCounter` — анімований лічильник

#### Бібліотеки:
- **recharts** — для побудови графіків (AreaChart, PieChart, BarChart)
- **react-markdown** + **remark-gfm** — для рендерингу AI-відповідей

#### Файли змінені:
1. `client/src/pages/admin-dashboard-page.tsx` — **НОВИЙ** — повна сторінка дашборду
2. `client/src/types/chat.ts` — додані типи для дашборду
3. `client/src/services/chat.service.ts` — додані методи `getDashboardStatistics` та `sendAdminMessage`
4. `client/src/routes/app-routes.tsx` — маршрут `/admin/dashboard` тепер використовує `AdminDashboardPage`

## 🧪 Тестування

### Передумови
1. Запустити backend: `dotnet run` в директорії `api/src/API`
2. Запустити frontend: `npm run dev` в директорії `client`
3. Увійти як адміністратор

### Тест 1: Завантаження дашборду
1. Перейти на `/admin/dashboard`
2. **Очікувано**: Сторінка завантажується з анімованими лічильниками та графіками
3. Перевірити, що всі 4 головні картки відображають правильні числа
4. Перевірити, що графіки відображають дані

### Тест 2: AI-чат — швидкі запити
1. На дашборді знайти секцію "AI Аналітик"
2. Натиснути кнопку "📊 Загальний звіт"
3. **Очікувано**: AI відповідає з детальним звітом, включаючи цифри та рекомендації
4. Натиснути кнопку "🔥 Критичні проблеми"
5. **Очікувано**: AI відповідає з інформацією про критичні проблеми, діаграми фільтруються

### Тест 3: AI-чат — довільні запити
1. Ввести в поле вводу: "Яка категорія найпроблемніша і чому?"
2. Натиснути кнопку відправки або Enter
3. **Очікувано**: AI дає детальну відповідь з аналізом категорій та рекомендаціями

### Тест 4: Фільтрація через AI
1. Ввести: "Покажи нові проблеми з дорогами"
2. **Очікувано**: AI відповідає і на дашборді з'являється тег фільтра
3. Діаграми відображають лише відфільтровані дані
4. Натиснути кнопку "X" біля тегу фільтра
5. **Очікувано**: Фільтр знімається, повертається повні дані

### Тест 5: Навігація
1. Натиснути на проблему в списку "Останні проблеми" або "Найвищий рейтинг"
2. **Очікувано**: Перехід на сторінку деталей проблеми

### Тест 6: Обробка помилок
1. Відключити backend і спробувати завантажити дашборд
2. **Очікувано**: Відображається повідомлення про помилку з іконкою

### Тест 7: Автооновлення
1. Відкрити дашборд і залишити вкладку відкритою на 2+ хвилини
2. Додати нову проблему з іншого акаунту
3. **Очікувано**: Через ~60 секунд дані на дашборді оновлюються

## 📋 Приклади запитів до AI-аналітика

| Запит | Очікуваний результат |
|-------|---------------------|
| "Дай загальний звіт" | Детальний статистичний звіт по всій системі |
| "Покажи критичні проблеми" | Аналіз критичних проблем + фільтр на дашборді |
| "Проаналізуй тренди" | Аналіз місячних трендів з прогнозами |
| "Яка категорія найпроблемніша?" | Порівняння категорій з рекомендаціями |
| "Оціни ефективність вирішення" | Аналіз рівня вирішення та часу |
| "Покажи нові проблеми з дорогами" | Фільтрація дашборду + аналіз |
| "Які рекомендації для покращення?" | Конкретні поради на основі даних |
| "Порівняй цей місяць з минулим" | Порівняльний аналіз двох періодів |

## 🔒 Безпека

- Всі ендпоінти адмін-дашборду захищені атрибутом `[Authorize(Roles = "Administrator")]`
- AI має доступ лише до агрегованих статистичних даних
- AI не має доступу до паролів, токенів або інших чутливих даних
- Токен аутентифікації передається через Clerk provider

## 🎨 Дизайн

Дизайн дашборду дотримується існуючих стилів проекту:
- **Кольори**: `#E42556` (акцент), `#1F2732` (темний), `#F7F9FD` (фон)
- **Типографіка**: Використовує `font-heading` для заголовків
- **Радіуси**: `rounded-2xl` для карток (як у design-system)
- **Тіні**: `shadow-sm` → `shadow-md` при hover
- **Анімації**: `transition-all`, `hover:-translate-y-0.5`, `animate-spin`, `animate-bounce`, `animate-pulse`
