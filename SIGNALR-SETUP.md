# SignalR Real-Time Features Setup

## Що було реалізовано

### Backend (API)

1. **SignalR Hubs**:
   - `CommentsHub` - для real-time оновлення коментарів
   - `NotificationsHub` - для системи нотифікацій

2. **Функціонал**:
   - Real-time оновлення коментарів при додаванні нового коментаря
   - Нотифікації автору проблеми про нові коментарі
   - Нотифікації про зміну статусу проблеми

3. **Файли**:
   - `api/src/API/Hubs/CommentsHub.cs`
   - `api/src/API/Hubs/NotificationsHub.cs`
   - `api/src/API/DTOs/Notifications/NotificationDto.cs`
   - `api/src/Application/Common/Interfaces/ISignalRService.cs`
   - `api/src/Infrastructure/Services/SignalRService.cs`

### Frontend (Client)

1. **SignalR Integration**:
   - SignalR сервіс для підключення до Hub'ів
   - SignalR контекст для глобального стану
   - Хук для real-time коментарів

2. **UI Компоненти**:
   - Дзвіночок нотифікацій у Header
   - Real-time оновлення коментарів у ProblemDetailPopup

3. **Файли**:
   - `client/src/services/signalr-service.ts`
   - `client/src/contexts/signalr-context.tsx`
   - `client/src/hooks/use-realtime-comments.ts`
   - `client/src/components/notifications/notifications-bell.tsx`

## Як це працює

### Real-time коментарі

1. Коли користувач відкриває деталі проблеми, він автоматично приєднується до групи цієї проблеми
2. Коли хтось додає коментар, він миттєво з'являється у всіх користувачів, які переглядають цю проблему
3. При закритті popup'а користувач виходить з групи

### Система нотифікацій

1. Користувач автоматично приєднується до своєї групи при вході в систему
2. Коли хтось коментує його проблему, він отримує нотифікацію
3. Коли змінюється статус його проблеми, він отримує нотифікацію
4. Нотифікації відображаються у дзвіночку в Header з лічильником непрочитаних

## Запуск проекту

### Backend

1. Переконайтеся, що у вас встановлено .NET 8 SDK
2. Перейдіть до директорії API:
   ```bash
   cd api/src/API
   ```
3. Запустіть проект:
   ```bash
   dotnet run
   ```

API буде доступний за адресою: `http://localhost:5000`

SignalR Hub'и будуть доступні за адресами:
- `http://localhost:5000/hubs/comments`
- `http://localhost:5000/hubs/notifications`

### Frontend

1. Переконайтеся, що у вас встановлено Node.js (версія 18+)
2. Перейдіть до директорії клієнта:
   ```bash
   cd client
   ```
3. Встановіть залежності (якщо ще не встановлені):
   ```bash
   npm install
   ```
4. Створіть файл `.env` на основі `.env.example`:
   ```bash
   cp .env.example .env
   ```
5. Переконайтеся, що у `.env` вказано правильний URL API:
   ```
   VITE_API_URL=http://localhost:5000
   ```
6. Запустіть проект:
   ```bash
   npm run dev
   ```

Клієнт буде доступний за адресою: `http://localhost:5173`

## Тестування

### Тест real-time коментарів

1. Відкрийте додаток у двох різних браузерах або вкладках
2. Увійдіть під різними користувачами
3. Відкрийте одну й ту ж проблему в обох вкладках
4. Додайте коментар в одній вкладці
5. Коментар має миттєво з'явитися в іншій вкладці

### Тест нотифікацій

1. Відкрийте додаток у двох різних браузерах або вкладках
2. Увійдіть під різними користувачами
3. Користувач A створює проблему
4. Користувач B додає коментар до проблеми користувача A
5. Користувач A має отримати нотифікацію (дзвіночок у Header)
6. При кліку на нотифікацію користувач переходить до списку проблем

## Troubleshooting

### SignalR не підключається

1. Переконайтеся, що backend запущений і доступний
2. Перевірте консоль браузера на наявність помилок
3. Переконайтеся, що CORS налаштований правильно в `Program.cs`
4. Перевірте, що токен JWT валідний

### Коментарі не оновлюються в real-time

1. Перевірте, що користувач успішно приєднався до групи проблеми
2. Перегляньте консоль браузера на наявність помилок SignalR
3. Переконайтеся, що `ISignalRService` правильно викликається в `CreateCommentCommand`

### Нотифікації не приходять

1. Перевірте, що користувач приєднався до своєї групи (перевірте консоль)
2. Переконайтеся, що `userId` правильно передається
3. Перевірте, що нотифікації відправляються в `CreateCommentCommand` та `UpdateProblemCommand`

## Додаткові можливості для розширення

1. **Збереження нотифікацій у базі даних** - зараз нотифікації зберігаються тільки в пам'яті клієнта
2. **Відповіді на коментарі** - можна додати можливість відповідати на коментарі з нотифікаціями
3. **Typing indicators** - показувати, коли хтось друкує коментар
4. **Online status** - показувати, хто зараз онлайн
5. **Read receipts** - показувати, хто прочитав коментар
