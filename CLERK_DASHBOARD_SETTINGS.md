# Оновлена інструкція для налаштування Clerk Dashboard

## Виконані зміни в коді

### 1. Синхронізація з Clerk Dashboard
Тепер при оновленні профілю користувача (ім'я, прізвище) або зміні ролі - ці зміни автоматично синхронізуються з Clerk Dashboard.

**Файли що були змінені:**
- `Application/Users/Commands/UpdateUserCommand.cs` - синхронізує ім'я та прізвище
- `Application/Users/Commands/ChangeRoleForUserCommand.cs` - синхронізує роль через public_metadata
- `Application/Users/Commands/DeleteUserCommand.cs` - видаляє користувача з Clerk
- `Application/Users/Commands/CreateUserCommand.cs` - створює користувача в Clerk при створенні адміністратором

### 2. Виправлено проблему з 401 для get-by-id
Endpoint `GET /problems/get-by-id/{id}` тепер доступний для анонімних користувачів (AllowAnonymous).

### 3. Локалізація на українську
Розширено файл `clerk-uk-UA.ts` з повною локалізацією для:
- Помилки входу ("Не вдалося знайти ваш акаунт", "Неправильний пароль")
- Тексти 2FA та email verification
- Тексти альтернативних методів входу
- Всі кнопки та підписи

### 4. Оновлено стилі форм логіну та реєстрації
Форми Clerk тепер займають 100% правої половини екрану.

### 5. Редірект після виходу
Після виходу з акаунту користувач перенаправляється на `/map`.

## Налаштування в Clerk Dashboard

### Отримання Secret Key

1. Перейдіть в **Clerk Dashboard** > **Configure** > **API Keys**
2. Скопіюйте **Secret Key** (починається з `sk_test_` або `sk_live_`)
3. Вставте його в `appsettings.json`:

```json
{
  "Clerk": {
    "Domain": "cool-snake-75.clerk.accounts.dev",
    "SecretKey": "sk_test_YOUR_ACTUAL_SECRET_KEY"
  }
}
```

**ВАЖЛИВО:** Ніколи не комітьте секретні ключі в Git! Використовуйте `appsettings.Development.json` або змінні середовища для локальної розробки.

---

### Вимкнення двофакторної автентифікації (2FA)

Якщо ви бачите "You're signing in from a new device. We're asking for verification to keep your account secure." при вході:

1. Перейдіть в **Clerk Dashboard**
2. Виберіть **Configure** > **Sessions**
3. Знайдіть розділ **Multi-factor** 
4. Виберіть **Never** (Ніколи не вимагати MFA)
5. Збережіть зміни

**Або для конкретного користувача:**
1. Перейдіть в **Users**
2. Знайдіть користувача
3. У вкладці **Security** вимкніть всі MFA методи

---

### Вимкнення верифікації email при реєстрації

Якщо ви хочете вимкнути підтвердження email для нових користувачів:

1. Перейдіть в **Clerk Dashboard**
2. Виберіть **Configure** > **Email, Phone, Username**
3. У розділі **Email address**:
   - Знайдіть опцію **Require verification** 
   - Вимкніть її (toggle off)
4. Збережіть зміни

**Примітка:** Вимкнення верифікації email знижує безпеку акаунтів. Рекомендуємо залишити увімкненим для production.

---

### Вибір акаунту при вході (Account Switcher)

Щоб користувач мав можливість вибрати існуючий акаунт або використати інший:

1. Перейдіть в **Clerk Dashboard**
2. Виберіть **Configure** > **Sessions**
3. У розділі **Multi-session handling**:
   - Виберіть **Multi-session** (замість Single session)
4. Збережіть зміни

Тепер при вході користувач зможе:
- Продовжити в існуючий акаунт
- Або додати новий акаунт

---

### Коли адміністратор створює користувача

При створенні користувача адміністратором через API:

1. Користувач автоматично створюється в Clerk з паролем
2. Email **НЕ вимагає** верифікації (користувач одразу може входити)
3. Роль автоматично встановлюється в `public_metadata`

Користувач отримає email з повідомленням про створення акаунту та зможе увійти з вказаним паролем.

## Можливі проблеми та їх вирішення

### Помилка "form_identifier_not_found"
**Причина:** Користувач з таким email не існує в Clerk
**Рішення:** Зареєструвати нового користувача або перевірити правильність email

### Помилка "form_password_incorrect"  
**Причина:** Неправильний пароль
**Рішення:** Ввести правильний пароль або скинути через "Забули пароль?"

### 401 Unauthorized при запитах до API
**Причина:** Токен відсутній або недійсний
**Рішення:** Переконатися що користувач залогінений та токен передається в заголовку Authorization

### Помилка синхронізації з Clerk
**Причина:** SecretKey не налаштований
**Рішення:** Додати правильний SecretKey в appsettings.json

## Тестування

Щоб перевірити що все працює:

1. **Тест входу:** Спробуйте увійти з правильним/неправильним паролем - повинні з'являтися українські повідомлення про помилки

2. **Тест синхронізації:** 
   - Оновіть профіль користувача в додатку
   - Перевірте в Clerk Dashboard що дані оновились

3. **Тест видалення:**
   - Видаліть користувача в додатку
   - Перевірте що він видалений і в Clerk Dashboard

4. **Тест виходу:**
   - Вийдіть з акаунту
   - Перевірте що вас перенаправило на `/map`

5. **Тест перегляду проблем:**
   - Без авторизації відкрийте `/problems/get-by-id/{id}`
   - Повинно працювати без 401
