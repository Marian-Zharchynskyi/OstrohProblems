# AI Integration Implementation Plan - Ostroh Problems

## 📋 Огляд

Цей документ описує інтеграцію AI (Google Gemini) в проект Ostroh Problems для надання інтелектуального чат-асистента користувачам платформи.

---

## ✅ Що реалізовано

### 1. Backend (API)

#### Нові файли:
- `api/src/Application/Services/GeminiService/IGeminiService.cs` - інтерфейс сервісу
- `api/src/Infrastructure/Services/GeminiService.cs` - реалізація Gemini AI сервісу
- `api/src/API/Controllers/ChatController.cs` - контролер для чат-запитів
- `api/src/API/DTOs/Chat/ChatDtos.cs` - DTO для чату

#### Модифіковані файли:
- `api/src/Infrastructure/ConfigureInfrastructure.cs` - реєстрація GeminiService
- `api/src/Application/Common/Interfaces/Queries/IRatingQueries.cs` - додано GetByProblemId
- `api/src/Infrastructure/Persistence/Repositories/RatingRepository.cs` - реалізація GetByProblemId
- `api/src/API/appsettings.json` - додано секцію Gemini конфігурації

#### Функціональність API:
- **Класифікація намірів** - AI автоматично визначає тип запиту користувача
- **Контекстні відповіді** - різні відповіді залежно від ролі користувача
- **Запити до БД** - AI може витягувати реальні дані про проблеми
- **Безпека даних** - AI не має доступу до чутливих даних (паролі, токени)
- **Транскрипція голосу** - Gemini нативно обробляє аудіо (multimodal) без додаткових API
- **AI-екстракція даних проблеми** - автоматичне визначення категорії, пріоритету, вулиці з опису

### 2. Frontend (Client)

#### Нові файли:
- `client/src/types/chat.ts` - типи для чату
- `client/src/services/chat.service.ts` - сервіс для API запитів
- `client/src/components/chat/ai-chat.tsx` - UI компонент чату
- `client/src/pages/create-issue-ai-page.tsx` - сторінка AI-assisted створення проблеми
- `client/src/components/shared/create-problem-dropdown.tsx` - dropdown для вибору способу створення

#### Модифіковані файли:
- `client/src/components/shared/layout.tsx` - інтеграція чату в layout
- `client/src/components/shared/header.tsx` - dropdown замість прямого посилання
- `client/src/routes/app-routes.tsx` - новий роут `/problems/create-ai`

#### Функціональність UI:
- **Плаваюча кнопка** - завжди видима в правому нижньому куті
- **Анімації** - плавне відкриття/закриття чату
- **Голосовий ввід** - запис голосу через мікрофон з прямою обробкою через Gemini (без проміжної транскрипції)
- **Індикатор прогресу** - показує "Розпізнаю голос..." під час обробки
- **Підтримка української мови** - повна підтримка розпізнавання та відповідей українською
- **Відображення проблем** - картки проблем прямо в чаті з можливістю переходу
- **Адаптивний дизайн** - працює на різних розмірах екрану
- **AI-створення проблем** - окрема сторінка з чатом та preview полів
- **Dropdown вибору** - користувач може обрати спосіб створення (вручну/AI)

## 🔧 Конфігурація

### API Key
Додайте ваш Gemini API ключ в `appsettings.json`:
```json
{
  "Gemini": {
    "ApiKey": "YOUR_ACTUAL_GEMINI_API_KEY"
  }
}
```

### Моделі Gemini
Використовуються наступні моделі:
- **gemini-2.0-flash** - основна "робоча конячка" для чату, аналізу скарг та роботи з фото.
- **gemini-2.0-flash-lite-001** - полегшена швидка модель для класифікації намірів та простих перевірок.
- **gemini-2.5-pro** - найрозумніша модель для складних адміністративних задач (з обережністю через ліміти).
- **text-embedding-004** - модель для векторного пошуку (RAG) та перетворення тексту в вектори.

## 🔒 Безпека даних

AI **НЕ має доступу** до:
- Хешів паролів користувачів
- Refresh токенів
- ClerkId
- Внутрішніх системних даних

AI **має доступ** до:
- Публічної інформації про проблеми
- Категорій, статусів, пріоритетів
- Коментарів та рейтингів
- Імен користувачів (без email)

## 🚀 Можливі покращення

### Короткострокові:
1. **Історія чату** - збереження в localStorage або БД
2. **Streaming відповідей** - показ відповіді по мірі генерації
3. **Більше фільтрів** - пошук за датою, локацією, автором
4. **Покращення точності** - fine-tuning моделі розпізнавання голосу для українського акценту

### Середньострокові:
1. ~~**Створення проблем через чат**~~ ✅ **РЕАЛІЗОВАНО** - AI-assisted створення проблем
2. **Аналітика** - "Скільки проблем вирішено цього місяця?"
3. **Рекомендації** - "Які проблеми схожі на мою?"
4. **Мультимовність** - підтримка англійської мови

### Довгострокові:
1. **RAG (Retrieval Augmented Generation)** - пошук по всій БД з векторним індексом
2. **Fine-tuning** - навчання моделі на специфічних даних проекту
3. **Автоматична категоризація** - AI визначає категорію нової проблеми
4. **Sentiment Analysis** - аналіз тональності коментарів
5. **Прогнозування** - передбачення часу вирішення проблеми
6. **Автоматичні сповіщення** - AI інформує про важливі оновлення

## 🆕 AI-Assisted Створення Проблем (v1.1)

### Опис функціоналу:
Користувачі можуть створювати проблеми через AI-асистента замість ручного заповнення форми.

### Як це працює:
1. Користувач натискає "Подати проблему" в header
2. З'являється dropdown з двома варіантами:
   - **Вручну** - перехід на звичайну форму `/problems/create`
   - **З AI асистентом** - перехід на AI-сторінку `/problems/create-ai`
3. На AI-сторінці користувач описує проблему текстом або голосом
4. AI автоматично визначає:
   - Заголовок проблеми
   - Опис
   - Категорію (Дороги, Освітлення, Сміття тощо)
   - Пріоритет (Низький, Середній, Високий, Критичний)
   - Назву вулиці (якщо згадана)
5. Дані відображаються в preview-панелі справа
6. Користувач може редагувати будь-яке поле
7. Користувач вибирає місце на карті та додає фото (опціонально)
8. Натискає "Створити проблему"

### API Endpoints:
```
POST /chat/extract-problem
Content-Type: application/json
Authorization: Bearer {token}
Body: { "message": "опис проблеми" }

POST /chat/extract-problem-voice
Content-Type: multipart/form-data
Authorization: Bearer {token}
Body: audioFile (WebM)
```

### Відповідь:
```json
{
  "title": "Яма на дорозі",
  "description": "Велика яма біля парку Шевченка...",
  "categories": ["Дороги"],
  "priority": "Високий",
  "streetName": "вул. Шевченка",
  "latitude": null,
  "longitude": null,
  "aiMessage": "Зрозумів! Ви повідомляєте про яму на дорозі...",
  "isComplete": true
}
```
