# SignalR - Інструкція по використанню

## Що було додано

✅ **Real-time оновлення коментарів** - коментарі з'являються миттєво у всіх користувачів без перезавантаження сторінки

✅ **Система нотифікацій** - дзвіночок у Header, який показує:
- Нові коментарі до ваших проблем
- Зміну статусу ваших проблем
- Лічильник непрочитаних нотифікацій

## Швидкий старт

### 1. Backend (API)

```bash
cd api/src/API
dotnet run
```

### 2. Frontend (Client)

```bash
cd client
npm install
npm run dev
```

## Як перевірити, що все працює

### Перевірка real-time коментарів:

1. Відкрийте сайт у **двох різних вкладках браузера**
2. Увійдіть під **різними користувачами** в кожній вкладці
3. Відкрийте **одну й ту ж проблему** в обох вкладках
4. Додайте коментар в **першій вкладці**
5. ✨ Коментар **миттєво з'явиться** у другій вкладці!

### Перевірка нотифікацій:

1. Відкрийте сайт у **двох різних вкладках**
2. **Користувач A** створює нову проблему
3. **Користувач B** додає коментар до проблеми користувача A
4. ✨ У **користувача A** з'явиться **червоний кружечок** на дзвіночку в Header
5. Натисніть на дзвіночок - побачите нотифікацію про новий коментар

## Технічні деталі

### Backend:
- **CommentsHub** (`/hubs/comments`) - для коментарів
- **NotificationsHub** (`/hubs/notifications`) - для нотифікацій
- Автоматична відправка нотифікацій при створенні коментарів та зміні статусів

### Frontend:
- **SignalRProvider** - глобальний контекст для SignalR
- **NotificationsBell** - компонент дзвіночка в Header
- **useRealtimeComments** - хук для real-time коментарів

## Що відбувається "під капотом"

### Коли ви додаєте коментар:

1. 📤 Коментар відправляється на сервер через звичайний API запит
2. 💾 Сервер зберігає коментар у базі даних
3. 📡 Сервер через SignalR відправляє коментар всім, хто зараз дивиться цю проблему
4. 🔔 Сервер відправляє нотифікацію автору проблеми
5. ✨ Коментар миттєво з'являється у всіх користувачів

### Коли змінюється статус проблеми:

1. 📤 Запит на зміну статусу відправляється на сервер
2. 💾 Сервер оновлює статус у базі даних
3. 🔔 Сервер відправляє нотифікацію автору проблеми
4. ✨ Автор бачить нотифікацію у дзвіночку

## Важливо!

- SignalR автоматично підключається при вході в систему
- SignalR автоматично переподключається при втраті з'єднання
- Нотифікації зберігаються тільки в пам'яті браузера (при перезавантаженні сторінки зникають)

## Можливі проблеми та рішення

### Коментарі не оновлюються в real-time

**Рішення**: 
- Перевірте консоль браузера (F12) на наявність помилок
- Переконайтеся, що backend запущений
- Перевірте, що ви увійшли в систему

### Нотифікації не приходять

**Рішення**:
- Переконайтеся, що ви увійшли в систему
- Перевірте консоль браузера на помилки
- Спробуйте перезавантажити сторінку

### SignalR не підключається

**Рішення**:
- Переконайтеся, що backend запущений на `http://localhost:5000`
- Перевірте файл `.env` у папці `client` - має бути `VITE_API_URL=http://localhost:5000`
- Перевірте, що ваш JWT токен валідний
